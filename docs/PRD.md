# MixClaw 项目需求文档 (PRD)

## 1. 项目愿景
打造一个结合了 **PicoClaw 极致轻量 (Go 语言, 极低内存, 毫秒级启动)** 与 **OpenClaw 丰富交互 (现代化全功能 Web UI, 多 Agent 协作, 强大 Skills 系统)** 的下一代单文件 AI Agent 框架。

## 2. 核心架构设计
* **后端语言**: Go (Golang) - 保证编译后为单一二进制文件，跨平台，零运行时依赖。
* **前端栈**: 原生 HTML/CSS/JS (或无编译依赖框架) - 编译时直接嵌入 Go 二进制文件 (使用 `go:embed`)，无需额外启动 Node.js 或前端服务器。
* **数据存储**: 本地纯文本/JSONL，无需部署数据库服务。

## 3. 核心功能需求

### 3.1 现代化 Web UI
* **开箱即用**: 启动二进制文件即可在浏览器访问，无需繁琐部署。
* **多会话界面**: 支持 Markdown、代码高亮、流式打字机效果聊天。
* **可视化配置**: 提供直观界面配置大模型 Provider (如火山引擎)、模型参数、以及代理的提示词。
* **本地文件与记忆管理**: 侧边栏支持文件树视图，可以浏览、编辑工作区文件及记忆 (MEMORY.md)。
* **多 Agent 面板**: 可视化查看和管理主代理及其派生的子代理状态。

### 3.2 强大的 Agent 核心
* **全模型支持**: 原生支持火山引擎 (Volcengine/Coding Plan)、DeepSeek、OpenAI 及各种本地模型 (vLLM、Ollama)。
* **丰富的工具系统**:
  * 核心内置：终端命令沙箱执行、文件读写、网页内容获取与搜索。
  * 扩展技能：类似 OpenClaw 的社区 Skills 扩展机制，一键安装。
* **Agent 协作机制**: 主代理遇到子任务时，可通过 Spawn 工具衍生专门任务的子代理，并行工作并收集汇总。

### 3.3 轻量化与安全标准
* **启动速度**: < 100ms 级别。
* **内存占用**: 稳态运行内存 < 30MB。
* **受控执行**: 限制读取和操作的目录以保障宿主机安全，命令执行具备黑白名单拦截功能。

## 4. 里程碑规划
* **Phase 1 (基础框架与对话)**: 构建 Go 核心 API + 内嵌单文件 Web UI，打通基础对话流程及配置存储。
* **Phase 2 (工具链与记忆)**: 集成并实现 Tools 调用逻辑，增加前端文件树管理与长期记忆存储交互。
* **Phase 3 (多代理调度)**: 研发多 Agent (Subagent) Spawn 及协调机制，实现复杂任务自主拆分与并行处理。
